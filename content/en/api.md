## API Guide

This document describes the authentication, endpoints, requests, and responses needed to use the sDB API.

## Getting Started

To use the API, you need an issued API key and the production API base URL. If you do not have an API key, use the portfolio Contact page for the API-key acquisition flow:

- [https://takumi-tokunaga.com/contact/](https://takumi-tokunaga.com/contact/)

Use the API base URL issued with your API key. The examples below use a placeholder:

```text
https://api.sdb.takumi-tokunaga.com
```

## Authentication

`/v1/*` endpoints generally require an API key. Send it with this header:

```http
X-API-Key: <issued-key>
Accept: application/json
```

For JSON `POST` requests, also send:

```http
Content-Type: application/json
```

## Common Contract

- Encoding is UTF-8.
- Responses are JSON.
- Entity IDs are exposed as `publicId`. These are public IDs generated by sDB and are not official MEXT identifiers.
- List endpoints use `limit` and `offset`.
- `prefectureCode` uses values such as `JP-13`.

Example error:

```json
{
  "error": "API_KEY_INVALID",
  "message": "api key is invalid"
}
```

Status code guide:

- `400`: invalid query parameters or request body
- `401`: missing or invalid API key
- `402`: daily credits are insufficient
- `403`: API key is revoked or access is forbidden
- `404`: requested `publicId` not found
- `500`: internal server error

## Institutions

### `GET /v1/institutions`

Search institutions.

```bash
curl "https://api.sdb.takumi-tokunaga.com/v1/institutions?q=ritsumeikan&type=university&prefectureCode=JP-26&limit=10" \
  -H "X-API-Key: <issued-key>" \
  -H "Accept: application/json"
```

### `GET /v1/institutions/{publicId}`

Get one institution by public ID.

### `GET /v1/institutions/{publicId}/faculties`

List faculties or graduate schools for one institution.

### `GET /v1/faculties/{publicId}/departments`

List departments for one faculty.

## Suggest

### `GET /v1/suggest`

Return lightweight institution suggestions.

## Selection Events

### `POST /v1/selections`

Record that a user selected an institution, faculty, or department. This optional event can be used to improve search quality or ranking.

## Source And Processing Notice

API responses may contain data derived from public MEXT materials. If you display, store, redistribute, or use API response content in another service, show source attribution, describe the processing, link to the MEXT terms, and state the disclaimer.
