# API Documentation

This document describes the authentication, endpoints, requests, and responses needed to use the sDB API.

## Getting Started

To use the API, you need an issued API key and the production API base URL. If you do not have an API key, use the portfolio Contact page for the API-key acquisition flow:

- [https://takumi-tokunaga.com/contact/](https://takumi-tokunaga.com/contact/)

Use the API base URL issued with your API key. The examples below use a placeholder:

```text
https://api.sdb.takumi-tokunaga.com
```

## Authentication

`/v1/*` endpoints generally require an API key. Send it with this header:

```http
X-API-Key: <issued-key>
Accept: application/json
```

For JSON `POST` requests, also send:

```http
Content-Type: application/json
```

## Common Contract

- Encoding is UTF-8.
- Responses are JSON.
- Entity IDs are exposed as `publicId`. These are public IDs generated by sDB and are not official MEXT identifiers.
- List endpoints use `limit` and `offset`.
- `prefectureCode` uses values such as `JP-13`.

Example error:

```json
{
  "error": "API_KEY_INVALID",
  "message": "api key is invalid"
}
```

Status code guide:

- `400`: invalid query parameters or request body
- `401`: missing or invalid API key
- `402`: daily credits are insufficient
- `403`: API key is revoked or access is forbidden
- `404`: requested `publicId` not found
- `500`: internal server error

## Institutions

### `GET /v1/institutions`

Search institutions.

Query parameters:

- `q`: search text
- `type`: `university`, `graduate_school`, `junior_college`, `technical_college`, `technical_college_advanced`, `high_school`, or `vocational_school`
- `prefectureCode`: prefecture code such as `JP-13`
- `limit`: number of items to return
- `offset`: pagination offset

Request:

```bash
curl "https://api.sdb.takumi-tokunaga.com/v1/institutions?q=ritsumeikan&type=university&prefectureCode=JP-26&limit=10" \
  -H "X-API-Key: <issued-key>" \
  -H "Accept: application/json"
```

Response:

```json
{
  "items": [
    {
      "publicId": "example1",
      "name": "Example University",
      "displayName": "Example University",
      "institutionType": "university",
      "prefectureCode": "JP-13"
    }
  ],
  "total": 1,
  "limit": 10,
  "offset": 0
}
```

### `GET /v1/institutions/{publicId}`

Get one institution by public ID.

```bash
curl "https://api.sdb.takumi-tokunaga.com/v1/institutions/example1" \
  -H "X-API-Key: <issued-key>" \
  -H "Accept: application/json"
```

### `GET /v1/institutions/{publicId}/faculties`

List faculties or graduate schools for one institution.

```bash
curl "https://api.sdb.takumi-tokunaga.com/v1/institutions/example1/faculties?limit=20" \
  -H "X-API-Key: <issued-key>" \
  -H "Accept: application/json"
```

Response:

```json
{
  "items": [
    {
      "publicId": "faculty1",
      "displayName": "Example Faculty",
      "academicField": "engineering"
    }
  ],
  "total": 1,
  "limit": 20,
  "offset": 0
}
```

## Suggest

### `GET /v1/suggest`

Return lightweight institution suggestions.

Query parameters:

- `q`: search text
- `limit`: number of items to return

```bash
curl "https://api.sdb.takumi-tokunaga.com/v1/suggest?q=tokyo&limit=10" \
  -H "X-API-Key: <issued-key>" \
  -H "Accept: application/json"
```

## Selection Events

### `POST /v1/selections`

Record that a user selected an institution, faculty, or department. This optional event can be used to improve search quality or ranking. It is not required for read-only search usage.

Request:

```bash
curl "https://api.sdb.takumi-tokunaga.com/v1/selections" \
  -X POST \
  -H "X-API-Key: <issued-key>" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"type":"institution","publicId":"example1"}'
```

Body:

```json
{
  "type": "institution",
  "publicId": "example1"
}
```

`type` is one of `institution`, `faculty`, or `department`.
